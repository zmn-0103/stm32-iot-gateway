# STM32F407 晶振问题排查记录

## 问题现象

串口（USART3, 9600 baud）输出乱码，收到非 ASCII 字符（如 "噜噜"、"帱郳" 等）。

## 硬件环境

| 项目 | 值 |
|------|-----|
| MCU | STM32F407VET6 (LQFP100) |
| HSE 晶振 | **8 MHz**（实际板载） |
| USB 转串口 | CH340 |
| 串口引脚 | PB10 (TX), PB11 (RX) |
| 调试器 | CMSIS-DAP (SWD) |

## 根因

**CubeMX .ioc 文件中 HSE_VALUE 配置为 25MHz，但实际板载晶振是 8MHz。**

PLL 参数按 25MHz 计算（PLLM=25），实际输入 8MHz，导致：

```
VCO 输入 = 8MHz / 25 = 320kHz（设计目标 1MHz）
VCO 输出 = 320kHz × 336 = 107.52MHz（设计目标 336MHz）
SYSCLK   = 107.52MHz / 2 = 53.76MHz（设计目标 168MHz）
```

系统时钟偏差 3.125 倍，所有外设时钟（包括 USART 波特率）全部错误。

## 排查过程

### 第一阶段：时钟配置

原始代码为绕过 HSE 问题，临时用 HSI 16MHz：
```c
/* 暂时强制 HSI 16MHz, 先跑通外设, 再排查 HSE */
RCC_OscInitStruct.OscillatorType = RCC_OSCILLATORTYPE_HSI;
RCC_OscInitStruct.PLL.PLLState = RCC_PLL_OFF;
```

修改为 HSE + PLL 配置（但 PLLM 仍为 25，基于错误的 25MHz 假设）。

### 第二阶段：串口乱码

恢复 HSE+PLL 后串口仍乱码。LED 快闪确认 PLL 锁定（168MHz），但串口数据不可读。

尝试了多种排查手段：
- 手动写 BRR 寄存器 → 无效
- 发送固定字节 0x55 → 收到中文字符
- 发送 ASCII 字符串 "HELLO" → 仍为乱码
- 不同波特率（9600/115200）→ 均为乱码

### 第三阶段：错误的 USART 时钟假设

错误认为 USART 时钟享受 2×倍频（与定时器相同），手动写入：
```c
USART3->BRR = (uint16_t)(84000000UL / 9600UL);  // 8750，错误！
```

实际 USART 时钟 = PCLK1 = 42MHz，BRR=8750 对应波特率 ≈ 4800，是 9600 的一半。

### 第四阶段：发现 HSE 频率错误

用户指出实际晶振为 8MHz，CubeMX 配置为 25MHz。

## 最终修复

### 1. `Core/Inc/stm32f4xx_hal_conf.h`

```c
// 修改前
#define HSE_VALUE    25000000U

// 修改后
#define HSE_VALUE    8000000U
```

### 2. `Core/Src/system_stm32f4xx.c`

```c
// 修改前
#define HSE_VALUE    ((uint32_t)25000000)

// 修改后
#define HSE_VALUE    ((uint32_t)8000000)
```

### 3. `Core/Src/main.c` — SystemClock_Config

```c
// 修改前 (PLLM=25, 基于 25MHz HSE)
RCC_OscInitStruct.PLL.PLLM = 25;

// 修改后 (PLLM=8, 基于 8MHz HSE)
RCC_OscInitStruct.PLL.PLLM = 8;
```

### PLL 计算验证

```
HSE = 8MHz
PLLM = 8   → VCO 输入 = 8/8 = 1MHz ✓
PLLN = 336 → VCO 输出 = 1×336 = 336MHz ✓
PLLP = 2   → SYSCLK = 336/2 = 168MHz ✓
PLLP = 4   → USB 时钟 = 336/4 = 84MHz
```

### 总线分频

```
SYSCLK = 168MHz
AHB  /1  → HCLK  = 168MHz
APB1 /4  → PCLK1 = 42MHz (USART3 时钟)
APB2 /2  → PCLK2 = 84MHz (SPI1, I2C1 时钟)
```

## 关键知识点

### STM32F4 外设时钟倍频规则

当 APBx prescaler > 1 时：

| 外设类型 | 时钟源 | 是否倍频 |
|----------|--------|----------|
| 定时器 (TIM) | PCLKx | ✅ 2×PCLKx |
| USART/UART | PCLKx | ❌ PCLKx |
| SPI/I2C | PCLKx | ❌ PCLKx |

因此 `HAL_RCC_GetPCLK1Freq()` 返回的 42MHz 就是 USART3 的实际时钟，HAL 计算 BRR=4375 → 精确 9600 baud 是正确的。

### CubeMX 重新生成代码的注意事项

每次从 CubeMX 重新生成代码后，必须检查：
1. `stm32f4xx_hal_conf.h` 中 HSE_VALUE 是否为 8000000
2. `system_stm32f4xx.c` 中 HSE_VALUE 是否为 8000000
3. `main.c` 中 PLLM 是否为 8

建议同步修改 `.ioc` 文件中的 `RCC.HSE_VALUE=8000000`，避免 CubeMX 覆盖。
